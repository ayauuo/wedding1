# USB 串口驅動安裝指南

## 重要說明

**Windows 10/11 沒有內建 USB 轉串口驅動**，必須手動安裝。但有些情況下，Windows Update 可能會自動下載驅動（需要網路連接）。

## 自動安裝驅動的方法

### 方法 1：使用 Windows Update（推薦，最簡單）

1. **連接 USB 轉串口設備**到電腦
2. 打開「**裝置管理員**」（Win + X → 裝置管理員）
3. 找到「**其他裝置**」或「**未識別的裝置**」
4. 右鍵點擊未識別的設備 → **更新驅動程式**
5. 選擇「**自動搜索驅動程式**」
6. Windows 會自動從 Windows Update 下載並安裝驅動（需要網路連接）

### 方法 2：手動下載並安裝驅動

如果 Windows Update 無法找到驅動，請根據設備類型下載對應的驅動：

#### CH340/CH341 驅動
- **下載連結**：https://www.wch.cn/download/CH341SER_EXE.html
- **安裝步驟**：
  1. 下載 `CH341SER.EXE`
  2. 執行安裝程式
  3. 點擊「安裝」
  4. 重新啟動電腦（建議）

#### FTDI 驅動
- **下載連結**：https://ftdichip.com/drivers/vcp-drivers/
- **安裝步驟**：
  1. 下載對應的驅動安裝程式
  2. 執行安裝程式
  3. 按照提示完成安裝

#### Prolific PL2303 驅動
- **下載連結**：https://www.prolific.com.tw/US/ShowProduct.aspx?p_id=225&pcid=41
- **注意**：請下載最新版本的驅動，舊版本可能與 Windows 10/11 不兼容

#### CP210x 驅動
- **下載連結**：https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers
- **安裝步驟**：下載並執行安裝程式

## 為什麼我的筆電不需要安裝驅動？

可能的原因：
1. **Windows Update 已自動下載**：如果筆電有網路連接，Windows Update 可能已經自動下載並安裝了驅動
2. **設備使用不同的連接方式**：某些紙鈔機可能使用 USB HID 或其他協議，不需要串口驅動
3. **驅動已預裝**：某些電腦製造商可能預裝了常見的 USB 串口驅動

## 檢查驅動是否已安裝

### 方法 1：檢查裝置管理員
1. 打開「裝置管理員」
2. 展開「**連接埠 (COM 和 LPT)**」
3. 如果看到類似「USB-SERIAL CH340 (COM3)」的項目，表示驅動已安裝

### 方法 2：檢查 COM 口
1. 打開「命令提示字元」或「PowerShell」
2. 執行：`[System.IO.Ports.SerialPort]::GetPortNames()`
3. 如果看到 COM 口列表（如 COM1, COM2, COM3），表示驅動已安裝

## 應用程式的自動檢測功能

本應用程式會自動：
1. **檢測 COM 口**：啟動時自動檢測可用的 COM 口
2. **檢測設備類型**：自動識別 USB 轉串口晶片類型（CH340、FTDI 等）
3. **提供安裝提示**：如果未找到 COM 口，會顯示友好的提示訊息和驅動下載連結
4. **打開裝置管理員**：可以一鍵打開裝置管理員，方便手動安裝驅動

## 故障排除

### 問題：連接設備後沒有出現 COM 口

**可能原因：**
1. 驅動未安裝
2. USB 線未正確連接
3. 設備故障

**解決方法：**
1. 檢查裝置管理員中是否有未識別的設備
2. 如果有，按照「方法 1」使用 Windows Update 安裝驅動
3. 如果沒有，檢查 USB 線和設備連接

### 問題：安裝驅動後仍然沒有 COM 口

**可能原因：**
1. 驅動版本不正確
2. 驅動與 Windows 版本不兼容
3. 設備被其他程式占用

**解決方法：**
1. 卸載舊驅動，重新安裝最新版本
2. 檢查是否有其他程式正在使用 COM 口
3. 重新啟動電腦

### 問題：Windows Update 找不到驅動

**解決方法：**
1. 手動下載對應的驅動程式
2. 在裝置管理員中選擇「瀏覽電腦上的驅動程式」
3. 指定驅動程式所在的位置

## 快速檢查清單

在部署到新電腦前：
- [ ] USB 轉串口設備已連接
- [ ] 檢查裝置管理員中是否有 COM 口
- [ ] 如果沒有 COM 口，使用 Windows Update 或手動安裝驅動
- [ ] 確認 COM 口號碼（如 COM1, COM2, COM3 等）
- [ ] 以管理員身份運行應用程式

## 技術說明

### 為什麼需要驅動？

USB 轉串口設備需要驅動程式來：
1. 讓 Windows 識別設備
2. 創建虛擬 COM 口
3. 實現 USB 協議到串口協議的轉換

### Windows 內建驅動

Windows 10/11 **沒有內建**常見的 USB 轉串口驅動（CH340、FTDI 等），但：
- Windows Update 可能會自動下載驅動（如果有網路連接）
- 某些設備製造商可能提供 Windows 認證的驅動，可以通過 Windows Update 獲取

### 替代方案

如果無法安裝驅動，可以考慮：
1. **使用原生串口**：如果電腦有原生串口（RS232），可以直接使用
2. **使用 USB HID 設備**：某些紙鈔機支援 USB HID 協議，不需要串口驅動
3. **使用網路轉串口**：某些設備支援網路連接，可以通過 TCP/IP 通訊
