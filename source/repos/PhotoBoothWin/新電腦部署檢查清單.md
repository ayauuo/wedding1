# 新電腦部署檢查清單

## ✅ 必須安裝的項目

### 1. WebView2 Runtime（必須）

**為什麼需要：**
- 應用程式使用 WebView2 來顯示前端界面
- 這是 Microsoft Edge WebView2 的運行時組件

**如何安裝：**

#### 方法 1：自動安裝（推薦）
- Windows 10/11 的較新版本通常已包含
- 如果沒有，Windows Update 會自動安裝

#### 方法 2：手動安裝
1. 下載 WebView2 Runtime：
   - 官方下載：https://developer.microsoft.com/microsoft-edge/webview2/
   - 選擇「Evergreen Runtime」版本
   - 下載對應的安裝程式（通常是 `MicrosoftEdgeWebView2RuntimeInstaller.exe`）

2. 在新電腦上執行安裝程式
3. 安裝完成後重新啟動應用程式

**檢查是否已安裝：**
- 打開「設定」→「應用程式」→「已安裝的應用程式」
- 搜尋「WebView2」或「Microsoft Edge WebView2 Runtime」
- 如果看到相關項目，表示已安裝

### 2. USB 串口驅動（如果使用 USB 轉串口）

**為什麼需要：**
- 如果紙鈔機是透過 USB 轉串口連接，需要對應的驅動程式

**如何安裝：**
1. 查看 USB 轉串口設備的品牌和型號
2. 常見品牌：
   - **CH340/CH341**：下載 CH340 驅動
   - **FTDI**：下載 FTDI 驅動
   - **Prolific**：下載 PL2303 驅動
   - **CP210x**：下載 Silicon Labs 驅動

3. 下載對應的驅動程式並安裝
4. 安裝後，在「裝置管理員」中應該能看到 COM 口

**檢查方法：**
- 打開「裝置管理員」（Win + X → 裝置管理員）
- 展開「連接埠 (COM 和 LPT)」
- 應該能看到類似「USB-SERIAL CH340 (COM3)」的項目

## ❌ 不需要安裝的項目

### ✅ .NET Runtime
- **不需要**：已使用自包含部署，所有 .NET 依賴都已包含在 exe 中

### ✅ Visual C++ Redistributable
- **通常不需要**：自包含部署會包含必要的運行時

### ✅ 其他開發工具
- **不需要**：只需要運行打包好的 exe 即可

## 📋 完整部署步驟

### 步驟 1：準備新電腦
1. 確保 Windows 10/11 已更新到最新版本
2. 檢查是否已安裝 WebView2 Runtime（見上方說明）

### 步驟 2：安裝 WebView2 Runtime（如果沒有）
1. 下載並安裝 WebView2 Runtime
2. 重新啟動電腦（建議）

### 步驟 3：安裝 USB 串口驅動（如果需要）
1. 連接 USB 轉串口設備
2. 安裝對應的驅動程式
3. 在裝置管理員中確認 COM 口號

### 步驟 4：部署應用程式
1. 將整個 `publish` 資料夾複製到新電腦
2. 建議放在：`C:\PhotoBooth\` 或 `D:\PhotoBooth\`
3. 不要放在需要管理員權限的系統資料夾

### 步驟 5：首次運行
1. **以管理員身份運行** `PhotoBoothWin.exe`
   - 右鍵點擊 exe →「以系統管理員身分執行」
   - 串口訪問需要管理員權限

2. 檢查是否正常啟動
   - 應該能看到應用程式界面
   - 如果出現錯誤，查看 `Logs` 資料夾中的日誌

3. 檢查紙鈔機連接
   - 查看 `Logs/BillAcceptor_YYYYMMDD.log` 日誌文件
   - 應該能看到「紙鈔機已連接」的訊息

## 🔍 故障排除

### 問題 1：應用程式無法啟動

**可能原因：**
- 缺少 WebView2 Runtime

**解決方法：**
1. 安裝 WebView2 Runtime（見上方說明）
2. 重新啟動應用程式

### 問題 2：界面顯示空白

**可能原因：**
- WebView2 Runtime 版本過舊
- 前端文件缺失

**解決方法：**
1. 更新 WebView2 Runtime 到最新版本
2. 確認 `web` 資料夾中的所有文件都已複製

### 問題 3：無法連接紙鈔機

**可能原因：**
- 沒有以管理員身份運行
- USB 驅動未安裝
- COM 口被其他程式占用
- 串口號碼不正確

**解決方法：**
1. 以管理員身份運行應用程式
2. 檢查 USB 驅動是否已安裝
3. 檢查裝置管理員中的 COM 口號
4. 查看 `Logs/BillAcceptor_YYYYMMDD.log` 日誌文件

### 問題 4：找不到 COM 口

**可能原因：**
- USB 驅動未安裝
- USB 線未連接
- USB 轉串口設備故障

**解決方法：**
1. 檢查 USB 線是否連接
2. 安裝對應的 USB 驅動
3. 嘗試更換 USB 線或 USB 轉串口設備

## 📝 快速檢查清單

在部署到新電腦前，確認以下項目：

- [ ] Windows 10/11 作業系統
- [ ] WebView2 Runtime 已安裝（或準備安裝）
- [ ] USB 串口驅動已安裝（如果使用 USB 轉串口）
- [ ] 已複製整個 `publish` 資料夾到新電腦
- [ ] 準備以管理員身份運行應用程式
- [ ] 紙鈔機硬體已正確連接

## 💡 建議

1. **建立安裝腳本**：可以建立一個批次檔，自動檢查並安裝必要的組件
2. **測試環境**：建議先在測試電腦上完整測試一次
3. **備份日誌**：保留第一次運行的日誌文件，方便問題診斷
4. **文件位置**：建議將應用程式放在固定的位置（如 `C:\PhotoBooth\`）
