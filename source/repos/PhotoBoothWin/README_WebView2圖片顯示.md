# WebView2 裡圖片變超大／與瀏覽器不一致

## C# 有沒有動到圖片？

**沒有。** C# 只做兩件事：

- **save_image**：把 Vue 傳來的 base64 解碼後**原樣寫檔**，沒有縮放、沒有改尺寸。
- **upload**：把 Vue 傳來的 base64 **原樣 POST 到你的 API**，沒有縮放、沒有改尺寸。

所以「推過來後圖片變超大」**不是 C# 造成的**，多半是下面兩種之一。

---

## 情況一：畫面上看起來超大（在 WPF 裡看很大，瀏覽器裡正常）

這是 **WebView2 的視窗／viewport／DPI** 和一般瀏覽器不一樣造成的。

- WPF 視窗是 **全螢幕**（`WindowState="Maximized"`），WebView2 的「視窗寬度」可能是 1920px 或更高。
- 一般瀏覽器你可能是 1280px 或更小的視窗。
- Vue 裡很多地方用 `width: 100%`、`100vw`，在 WebView2 裡就會依「全螢幕寬度」去算，圖片就會被拉得很大。

**可行做法（在 Vue / 前端做）：**

1. **限制 #app 或最外層寬度**  
   例如在 `main.css` 或根元件設 `max-width: 1280px`（或你要的寬度），並置中，讓版面不要隨全螢幕無限變寬。

2. **在 WebView2 裡用固定 viewport**  
   偵測到在 WebView2（例如 `window.chrome?.webview`）時，動態改 viewport，例如固定成 `width=1280`，讓排版和你在瀏覽器裡看到的一致。

3. **WPF 端（可選）**  
   若你不想全螢幕填滿，可把 WebView2 放在一個固定寬高的容器裡（例如 1280x720），但會影響你現在的全螢幕 kiosk 設計。

---

## 情況二：上傳／存檔的「檔案」很大（MB 很大或像素很高）

這是 **Vue 傳給 C# 的圖本來就很大**。

- Vue 用 `canvas.toDataURL('image/jpeg', 0.9)` 時，畫布多大，輸出的圖就多大像素。
- 若畫布是 1920x1080 或更高，存檔／上傳的圖就是那個解析度，檔案就會大。
- 在瀏覽器裡你看到的可能是用 CSS 縮小的「預覽」，但送給 C# 的 base64 仍是原始解析度。

**可行做法（在 Vue 做）：**

- 在呼叫 `toDataURL` 前，先把 canvas 縮小到你要的輸出尺寸（例如寬 1000px、或 1280px），再 `toDataURL`，再送給 C#。  
這樣 C# 收到的就是較小尺寸的圖，存檔／上傳的檔案就會變小，**C# 不用改**。

---

## 總結

| 現象 | 原因 | 誰要改 |
|------|------|--------|
| 在 WPF 裡「看起來」圖片超大 | WebView2 全螢幕、viewport／CSS 依大寬度排版 | Vue：限制 max-width / viewport |
| 上傳／存檔的「檔案」很大 | Vue 傳的 canvas 解析度就很大 | Vue：輸出前縮小 canvas 再 toDataURL |
| C# 有改圖嗎？ | 沒有，只原樣存檔／轉發 | 不用改 C# |

所以 **C# 沒有影響你的圖的尺寸**；要調整的是「在 WebView2 裡的版面」或「Vue 送出的解析度」。
